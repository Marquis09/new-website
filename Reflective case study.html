Project Details

  • My Role: UX Designer, Front-End Developer
  • Duration: 6 Weeks
  • Tools: VS Code, HTML/CSS/JS, GitHub
  • Target: Junior UX/Front-End Internship

1. Research & Strategic Context üéØ

My primary goal was to create a portfolio that not only looked professional but also demonstrated the core skills required for a Junior UX/Front-End role. This strategy was my compass throughout the development process.

Industry Alignment & Career Goals

I started by analyzing job descriptions for my target role. This validated my decision to hand-code the site using "CLEAN HTML/CSS/JS", as proving my technical competence in "FRONT-END DEVELOPMENT" is just as crucial as showcasing my "UX PROCESS". I knew a template or website builder would not achieve this level of technical proof, making the code itself a vital deliverable.

Information Architecture & Planning

I defined the "INFORMATION ARCHITECTURE (IA)" and layout using "ITERATIVE SKETCHES" and a formal "SITEMAP" before starting the development. This allowed me to continuously test the flow of the five-page structure (`index`, `projects`, `case study`, `contacts`, `journal`) against the user's expected journey (an employer's recruitment journey). Prioritizing a "MOBILE FIRST" foundation from the start ensured structural integrity across all devices.

Strategic Change

A key strategic decision was restructuring the site post-initial draft to integrate the "CORE OFFERINGS (SKILLS SUMMARY) section immediately after the hero banner. This critical adjustment ensures my core technical value proposition is communicated within the employer's crucial "FIVE-SECOND SCAN TIME", dramatically improving the site's effectiveness as a marketing tool.

2. Code-Based Design Execution üíª

Visual Communication & Design System

My creative concept‚Äîinspired by the juxtaposition of "PRECISE ENGENEERING" and "SMOOTH FLOW"‚Äîwas documented in a "BASIC DESIGN SYSTEM" and then translated into CSS. This system defined the visual rationale for all elements, ensuring consistency across all pages.

Dynamic CSS Interactions (Flow)

To implement the "FLOWING/GRITTY" dynamic experience, I used advanced CSS transformations, directly addressing the creative brief:

  • Link Flow Effect: All primary links utilize a `transform: scaleX(0)` pseudo-element with a smooth `transition` to create a signature underline that flows left-to-right on hover.
  • Project Card Movement: Project cards employ a `transform: scale(1.05) rotate(0.5deg)` on hover, adding a dynamic, tactile lift and movement.
  • Dark Mode: The persistent "DARK MODE TOGGLE" was implemented using JavaScript's `localStorage` and the `body.dark-mode` CSS class.

3. Critical Reflection & Iterative Development üîÑ

The final stage required critical self-reflection, incorporating feedback, and making specific, rigorous changes to the codebase and functionality.

Functional Rigour & Problem-Solving

A key technical challenge was ensuring reliable communication through the contact form. I chose to move beyond simple client-side JavaScript validation by integrating "FORMSPREE" (for robust backend processing) and a dedicated "JAVA SCRIPT FETCH API" handler. This achieved asynchronous submission with a custom loading spinner, demonstrating my capability to integrate third-party services for professional-grade functionality.

Final IA Refinement

During implementation, a key change was made to simplify the Information Architecture (IA). The initial plan for a five-page structure was refined into the final "THREE-LINK PRIMARY NAVIGATION" (`Home`, `Projects`, `Contact`). This change allowed me to launch the core function quickly and "CONSOLIDATED THE PROJECTS OVERVIEW ONTO THE HOME PAGE", ensuring all critical work is immediately visible without an extra click.

Feedback-Driven Iteration

"The navigation links lacked clear signposting and did not integrate well with the aggressive visual theme."

Final Changes Implemented

In response, I implemented the following adjustments to improve the "COMMUNICATION (VISUAL)" score:

  • Navigation Refinement: The primary navigation links were "RESTYLED AS PROMINENT BUTTONS" to improve discoverability and better match the structural, engineered aesthetic.
  • Code Quality: I performed a final review to confirm adherence to semantic HTML5 standards and focused on isolating functional logic (JavaScript) from presentation (CSS), improving "CODE MAINTAINABILITY".

Conclusion on Learning

This project demonstrated the necessity of the "SYNERGY BETWEEN STRATEGIC DESIGN AND TECHNICAL EXECUSION". My most valuable takeaway is the importance of "CODE MAINTAINABILITY" and the power of iterative development. Moving forward, I will dedicate more time to pre-code wireframing and explore CSS pre-processors.
